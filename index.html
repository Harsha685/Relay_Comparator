Timestamp accuracy — each log's timestamps are converted to elapsed seconds from its first sample, then compared sample-by-sample between Source A and B. The average drift is expressed as a percentage of the total run duration.

Analog value accuracy — the reading at each matched sample is compared between sources. The average absolute difference is normalized against the observed value range to produce a percentage.

Relay status accuracy — the share of matched samples where the relay state (ON/OFF, HIGH/LOW, TRUE/FALSE, 1/0) agrees between Source A and B.

Overall accuracy — the equal-weighted average of the three metrics above (Timestamp, Analog Value, and Relay Status), computed over the number of samples common to both files.

Column matching is case-insensitive and tolerant of misspelled or differently-worded headers — it looks for words like "time"/"date" for Timestamp, "analog"/"value"/"voltage"/"signal"/"pot…" for Analog Value, and "relay"/"status"/"state" for Relay Status, anywhere in each header.

Each chart plots the analog value trace with a shaded reference grid, and includes a colored strip beneath it showing the relay status (ON/OFF) timeline for that source, so all three compared fields are visible at a glance.

Tables initially show the first 50 rows per source; use "Show More" below the tables to reveal more in batches — accuracy metrics always reflect every matched sample, not just the rows currently displayed.
